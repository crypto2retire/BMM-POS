STEP 2 — Create app/schemas/sale.py with:
- CartItem: barcode, quantity
- SaleCreate: list of CartItems, payment_method (cash/card/split), cash_tendered (required if cash or split), receipt_email optional
- SaleItemResponse: item name, vendor name, quantity, unit_price, line_total
- SaleResponse: id, subtotal, tax_amount, total, payment_method, change_given, created_at, list of SaleItemResponse

STEP 3 — Create app/routers/sales.py with:
- GET /api/v1/sales/ — admin sees all sales, vendor sees only sales containing their items, filterable by date range
- GET /api/v1/sales/{id} — single sale detail with all line items
- GET /api/v1/sales/summary/today — admin only, returns total sales count, total revenue, total tax collected for today

STEP 4 — Create app/routers/pos.py with:
- POST /api/v1/pos/sale — the main checkout endpoint. Logic:
  1. Accept list of barcodes with quantities
  2. Look up each item, verify status is active, get active_price (respects sale dates)
  3. Calculate subtotal
  4. Calculate tax: for each item check is_tax_exempt, only tax non-exempt items at 5.5%
  5. Calculate total
  6. If payment_method is cash: verify cash_tendered >= total, calculate change
  7. Record sale to sales table
  8. Record each line item to sale_items table
  9. Mark each sold item status as sold in items table
  10. Add each line_total to the vendor's balance in vendor_balances table
  11. Return SaleResponse with change amount if cash

- POST /api/v1/pos/payment-callback — placeholder endpoint that returns 200 OK (for future Poynt integration)

STEP 5 — Wire sales and pos routers into app/main.py
