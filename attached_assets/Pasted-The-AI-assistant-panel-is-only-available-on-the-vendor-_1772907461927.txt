The AI assistant panel is only available on the vendor dashboard and items list pages. It needs to also be available inside the Add Item and Edit Item forms so vendors can get help while filling out the form — especially to analyze a photo and suggest a name and description.

STEP 1 — Add the assistant to the Add/Edit item form in frontend/vendor/items.html

The form is currently a modal. Add the following to that modal:

At the top of the form, just below the photo upload field, add an AI assist button:
- A small button labeled "✨ Describe with AI"
- Only visible after a photo has been selected (hidden by default, shown once a photo preview appears)
- Charcoal background (#38383B), white text, small size, square corners to match brand
- When clicked, it sends the uploaded photo to the assistant API with the message: "Analyze this item photo. Suggest a name, category, price range, and write a 2-3 sentence product description."

When the assistant responds:
- Parse the response and look for name, category, and description suggestions
- Show a small suggestion panel directly inside the form (not a separate chat panel) with:
  * Suggested name with a "Use this" button next to it
  * Suggested category with a "Use this" button
  * Suggested description with a "Use this" button
  * A "Use all" button that fills all three fields at once
- When "Use this" or "Use all" is clicked, fill the corresponding form field(s) with the suggestion
- The vendor can edit the filled values freely after they are applied
- Dismiss button (X) closes the suggestion panel without applying anything

STEP 2 — Add a mini assistant chat inside the form modal

Below the suggestion panel, add a small collapsible chat section labeled "💬 Ask the assistant":
- Collapsed by default, expands when clicked
- Shows a simple text input and Send button
- The chat is scoped to item help — the system context sent with each message should include: "The vendor is currently filling out the Add Item form."
- Responses appear directly in this mini chat area
- This lets vendors ask things like "what category should a vintage lamp go in?" or "how should I price a hand-sewn quilt?"
- Photo button also available here — same behavior as before, sends photo for analysis

STEP 3 — When the form is in Edit mode

If the vendor opens the Edit form for an existing item:
- Pre-populate the assistant context with the item's current name and category
- The "✨ Describe with AI" button is always visible in edit mode (not just after photo upload) so they can re-analyze an existing photo or get a better description
- If the item already has a photo, send the existing photo URL in the assistant message as text context: "This item is called [name] in the [category] category."

STEP 4 — Make sure the assistant floating button (💬) is still available on the items page

The floating button in the bottom-right corner should still work as before for general inventory management chat. The in-form assistant is separate and scoped to that item only.

STEP 5 — Update the assistant API endpoint in app/routers/assistant.py

Add an optional field to the request schema:
- form_context: optional string — if provided, prepend it to the system prompt as additional context

In AssistantChatRequest (app/schemas/assistant.py) add:
    form_context: Optional[str] = None

In the assistant router, if form_context is provided, add it to the system prompt:
    "Current context: " + form_context
